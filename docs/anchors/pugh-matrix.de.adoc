= Pugh-Matrix
:categories: strategic-planning
:roles: software-architect, team-lead, product-owner
[%collapsible]
====
Full Name:: Pugh-Entscheidungsmatrix (auch Pugh Controlled Convergence)

[discrete]
== *Kernkonzepte*:

Matrix-Layout:: Kriterien als ZEILEN, Alternativen als SPALTEN
Referenz (Baseline):: Eine Alternative wird als Referenz festgelegt — durchgehend mit 0/S bewertet
Relative Bewertung:: Jede Alternative wird gegen die Referenz bewertet: Besser (+), Gleich (S), Schlechter (−)
Summenzeilen:: Σ(+), Σ(−), Σ(S) und Nettowert pro Alternative am Ende der Matrix
Strukturierte Evaluation:: Systematischer Vergleich über mehrere Dimensionen
Team-Entscheidung:: Unterstützt Konsensbildung im Team
Hybridlösungen:: Stärken verschiedener Alternativen kombinieren
Iterative Verfeinerung:: Mehrere Runden zur Konvergenz auf die beste Lösung

Hauptvertreter:: Stuart Pugh ("Total Design", 1991)

[discrete]
== *Varianten* (als Qualifier verwenden für konsistente Ergebnisse):

Unweighted 3-Point Pugh Matrix:: Originalform. Default, wenn kein Qualifier angegeben wird.
Weighted 3-Point Pugh Matrix:: Kriterien tragen Wichtungsgewichte. Standard im Six-Sigma- / QFD-Kontext.
Weighted 5-Point Pugh Matrix:: Feinere Granularität für eng beieinanderliegende Alternativen.
Pugh Controlled Convergence:: Iterativer Mehrrunden-Prozess mit Elimination, Hybridbildung und Referenzrotation.

[discrete]
== *Wann einsetzen*:

* Mehrere tragfähige Alternativen existieren
* Entscheidungskriterien sind bekannt, aber Trade-offs unklar
* Das Team muss einen Konsens finden
* Architektur- oder Technologieauswahlentscheidungen
====
